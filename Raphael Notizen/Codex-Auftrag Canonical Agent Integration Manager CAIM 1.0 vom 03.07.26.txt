# Codex-Auftrag: Canonical Agent Integration Manager (CAIM) 1.0 vom 03.07.26

## Ziel

Implementiere den **Canonical Agent Integration Manager (CAIM) 1.0** als neuen Foundation-Baustein für Projekt Kontinuum.

Der CAIM soll die zentrale, kanonische Verwaltungsinstanz für interne, lokale und später externe KI-Agenten werden.

Er ist nicht selbst der Agent, sondern verwaltet:

* Agentenregistrierung
* Agentenfähigkeiten
* erlaubte Werkzeuge
* Berechtigungen
* Status
* Lebenszyklus
* Router-Anbindung
* Governance-Protokollierung
* sichere Übergabe an bestehende Agenten

## Grundprinzip

Der CAIM ist die **Single Source of Truth** für registrierte Agenten.

Bestehende Agenten wie:

* memory
* web_agent
* file_agent
* vision_agent
* git_agent
* code_agent
* learning_agent
* identity_manager
* canonical_memory_manager

sollen nicht ersetzt, sondern kanonisch registriert und sichtbar gemacht werden.

---

## Kanonisches Datenmodell

`canonical_agents.json` soll mindestens diese Struktur besitzen:

```json
{
  "schema_version": "1.0",
  "agents": [
    {
      "id": "agent_memory",
      "name": "memory",
      "type": "internal",
      "status": "active",
      "version": "1.0",
      "description": "Verwaltet Erinnerungsbefehle und Memory-Zugriffe.",
      "capabilities": [
        "memory.read",
        "memory.write",
        "memory.status"
      ],
      "allowed_tools": [],
      "governance_required": false,
      "read_only": false,
      "entrypoint": "kontinuum.agents.memory_agent.MemoryAgent"
    }
  ],
  "last_updated": "",
  "hash": ""
}
```

## Systemintegration

In `KontinuumSystem` einbinden:

```python
self.canonical_agent_integration_manager = CanonicalAgentIntegrationManager(
    self.path_tools,
    self.storage,
    self.identity_manager,
)
self.agent_config["canonical_agent_integration_manager"] = self.canonical_agent_integration_manager
```

Im Systemstatus ergänzen:

```python
"canonical_agent_integration_manager": self.canonical_agent_integration_manager.status()
```

## Router-/Orchestrator-Anbindung

Der CAIM soll zunächst read-only in das Routing eingebunden werden.

Ziel für Version 1.0:

* bestehende Agenten registrieren
* Status anzeigen
* Fähigkeiten auffindbar machen
* noch keine riskante automatische Fremdagenten-Ausführung

Wenn der Router später eine Fähigkeit benötigt, soll CAIM beantworten können:

```text
Welcher Agent besitzt capability X?
Ist dieser Agent aktiv?
Ist Governance erforderlich?
Darf dieser Agent ausgeführt werden?
```

## Initiale Agentenregistrierung

Beim ersten Start soll CAIM vorhandene Agenten aus `system.agents` oder `agent_config` registrieren können, ohne bestehende Agentenlogik zu verändern.

Mindestens aufnehmen:

* memory
* web_agent
* file_agent
* vision_agent
* git_agent
* code_agent
* learning_agent
* identity_manager
* canonical_memory_manager
* canonical_identity_manager
* canonical_artifacts
* canonical_architecture
* canonical_api_registry
* canonical_database

---

## Sicherheitsregel

Für CAIM 1.0 gilt:

Keine externen Agenten automatisch ausführen.

Externe Agenten dürfen nur registriert, beschrieben und als `experimental` oder `disabled` markiert werden.

Ausführung externer Agenten erst in CAIM 1.1 oder später.
