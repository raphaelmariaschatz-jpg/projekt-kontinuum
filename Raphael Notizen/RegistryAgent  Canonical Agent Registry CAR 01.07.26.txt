RegistryAgent / Canonical Agent Registry (CAR) 01.07.26

Codex-Auftrag: RegistryAgent und Canonical Agent Registry (CAR) für Projekt Kontinuum integrieren

Ziel:
Bitte einen RegistryAgent und darauf aufbauend eine Canonical Agent Registry (CAR) integrieren. Die CAR wird das zentrale kanonische Verzeichnis aller Agenten und ihrer Fähigkeiten. Neue Agenten sollen sich künftig über die Registry in das System integrieren lassen, ohne den Request Router oder andere Komponenten manuell erweitern zu müssen.

Grundsatz:
Die CAR ist die einzige kanonische Quelle für Agenteninformationen. Es darf keine parallelen Agentenlisten oder mehrfach gepflegte Registrierungen geben.

---

## 1. Architektur prüfen

Vor der Implementierung:

* bestehende Agentenstruktur analysieren
* Registry-, Router-, CAM-, CDE-, Governance- und Memory-Struktur berücksichtigen
* keine Parallelarchitektur erzeugen
* vorhandene Agenten automatisch übernehmen
* bestehende Funktionalität vollständig erhalten

---

## 2. RegistryAgent erstellen

Aufgabe:

Der RegistryAgent verwaltet die Canonical Agent Registry.

Er ist verantwortlich für:

* Agenten registrieren
* Agenten aktualisieren
* Agenten deaktivieren
* Agenten suchen
* Agenten validieren
* Agentenversionen verwalten
* Agentenstatus überwachen
* Fähigkeiten verwalten
* Abhängigkeiten erkennen
* Kompatibilität prüfen

Read-only für bestehende Agenten.

Neue Registrierungen nur über definierte Registrierungsfunktionen.

---

## 3. Canonical Agent Registry

Bitte eine kanonische Registry anlegen.

Beispielsweise:

canonical_agent_registry.json

oder

canonical_agent_registry.db

Je Agent mindestens:

* agent_id
* name
* version
* description
* category
* status
* capabilities
* supported_commands
* supported_filetypes
* supported_sources
* dependencies
* permissions
* input_types
* output_types
* health_status
* created_at
* updated_at
* owner
* canonical
* experimental
* deprecated

## Wichtige Regeln

* Nur eine kanonische Agent Registry.
* Keine parallelen Registrierungen.
* Keine automatische Aktivierung neuer Agenten.
* Keine automatische Löschung.
* Read-only zuerst.
* Vollständig kompatibel zu CAM, CDE und Governance.
* Erweiterbar für zukünftige Agenten ohne Änderungen an der Kernarchitektur.
