Codex-Auftrag:
K – Kontinuierliches Internet-Lernen mit Bandbreitenlimit

Ziel:
K soll, sobald es aktiv ist, kontrolliert aus dem Internet lernen können, dabei aber maximal ca. 10 % der verfügbaren Internet-Bandbreite verwenden.

Wichtig:
Diese Funktion darf das System nicht blockieren, keine Governance-Dateien verändern und keine lokalen Daten unkontrolliert überschreiben.

Anforderungen:

1. Continuous Learning Mode

Bitte einen optionalen Modus einführen:

continuous_internet_learning_enabled = true/false

Standard:
false

Wenn aktiviert:
- K darf im Hintergrund regelmäßig Internetquellen abrufen.
- Die GUI bleibt jederzeit bedienbar.
- Lernprozesse laufen asynchron / im Hintergrund.
- Bei Programmende wird der Lernprozess sauber gestoppt.

2. Bandbreitenbegrenzung

Ziel:
Maximal ca. 10 % der verfügbaren Bandbreite nutzen.

Umsetzung:
- keine parallelen aggressiven Downloads
- kleine Request-Batches
- feste Pausen zwischen Requests
- Timeout pro Anfrage
- maximale Datenmenge pro Minute definieren
- optional adaptive Drosselung

Falls die echte Bandbreite nicht zuverlässig messbar ist:
- konservativen Default verwenden, z. B.:
  max_download_mb_per_hour
  max_requests_per_minute
  min_delay_between_requests_seconds

Wichtig:
Keine Speedtests beim Start durchführen, außer ausdrücklich aktiviert.

3. Lernquellen

Nur erlaubte, seriöse und öffentlich zugängliche Quellen verwenden.

Quellenklassen:
- wissenschaftliche Veröffentlichungen
- Universitäten
- öffentliche Dokumentationen
- Behörden / offizielle Quellen
- technische Dokumentationen
- frei zugängliche Enzyklopädien

Nicht verwenden:
- Paywall-Umgehung
- illegale Quellen
- Login-geschützte Inhalte ohne Freigabe
- private Daten
- Massenscraping
- aggressive Crawler

4. Lernspeicherung

Internet-Inhalte dürfen nicht ungeprüft direkt in den Kernspeicher geschrieben werden.

Stattdessen:

- neue Funde zuerst in eine Quarantäne-/Review-Zone legen
- Quelle, URL, Abrufzeit, Hash und Kurzsummary speichern
- Dubletten erkennen
- erst nach Freigabe oder Regelprüfung in Memory/Knowledge übernehmen

Beispielpfad:

32_data/internet_learning_queue/
32_data/internet_learning_review/

5. GUI-Anzeige

In der GUI sichtbar machen:

Internet-Lernen:
- aus / aktiv / pausiert
- genutzter Modus
- letzte Quelle
- letzte Lernzeit
- Anzahl neuer Funde
- Bandbreitenlimit aktiv: ja
- aktuelles Limit: 10 %

Optional:
Button:
"Internet-Lernen pausieren"
"Internet-Lernen fortsetzen"

6. Sicherheits- und Governance-Rahmen

- Keine Änderung an canonical_governance_baseline_34_1.json
- Keine Änderung an phase3_stabilization_policy_34_1.json
- Keine automatische Veränderung von Canonical-Artefakten
- Keine stillen Memory-Änderungen ohne Provenienz
- Jede gelernte Information braucht Quelle/URL/Timestamp/Hash
- Fehler und blockierte Quellen protokollieren
- Bei fehlendem Internet ehrlich melden:
  "Internet-Lernen pausiert: Internet nicht verfügbar oder keine verwertbaren Quellen."

7. Konfiguration

Bitte eine eigene Konfigurationsdatei prüfen oder ergänzen:

24_config/internet_learning_policy_34_1.json

Inhalt:
- enabled
- bandwidth_limit_percent = 10
- max_requests_per_minute
- min_delay_between_requests_seconds
- max_download_mb_per_hour
- allowed_source_classes
- blocked_source_classes
- review_required = true
- write_to_memory_directly = false

8. Tests

Bitte Tests ergänzen für:

- Continuous Learning standardmäßig deaktiviert
- Aktivierung startet Hintergrundprozess
- Deaktivierung stoppt Hintergrundprozess
- Bandbreitenlimit wird respektiert
- keine direkten Memory-Schreibungen
- gelernte Inhalte landen zuerst in Review/Queue
- Quellenmetadaten werden gespeichert
- GUI bleibt bedienbar
- fehlendes Internet wird sauber gemeldet

9. Ergebnisbericht

Bitte am Ende berichten:

- welche Dateien geändert oder neu erstellt wurden
- ob echtes Bandbreitenmessen möglich ist
- welches konservative Limit verwendet wird
- ob Internet-Lernen aktivierbar ist
- wo neue Funde gespeichert werden
- welche Tests bestanden wurden

Rahmen:
Nicht-invasive Umsetzung.
Keine Governance-Baselines verändern.
Keine automatische Massendatenaufnahme.
Kein aggressives Crawling.
