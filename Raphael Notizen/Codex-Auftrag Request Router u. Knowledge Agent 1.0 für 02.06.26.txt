# Codex-Auftrag: Request Router & Knowledge Agent 1.0 für 02.06.26

## Ziel

Bitte implementiere einen zentralen **Request Router** mit integriertem **KnowledgeAgent**, der jede Benutzereingabe zunächst analysiert und anschließend automatisch den passenden Agenten auswählt.

Der Router wird damit zum zentralen Einstiegspunkt aller Benutzeranfragen.

---

# Ausgangsproblem

Aktuell werden viele Anfragen unnötig über lokale Suche oder Internet verarbeitet.

Beispielsweise:

**Benutzer**

> Was ist Quantendynamik?

Aktuelles Verhalten:

* lokale Suche
* Memory
* Internet
* keine sinnvolle Antwort

Gewünschtes Verhalten:

* Frage als Wissensfrage erkennen
* KnowledgeAgent auswählen
* Antwort aus vorhandenem Wissen erzeugen
* Internet nur ergänzend nutzen, wenn erforderlich

---

# Neue Architektur

```
Benutzereingabe
        │
        ▼
Request Router
        │
        ├── KnowledgeAgent
        ├── LearningAgent
        ├── WebAgent
        ├── FileAgent
        ├── MemoryAgent
        ├── StatusAgent
        ├── CanonicalEngine
        └── weitere Agenten
```

Der Router entscheidet ausschließlich anhand der Benutzerintention.

---

# Pflichtfunktionen

## 1. Intent Recognition

Vor jeder Verarbeitung muss die Eingabe klassifiziert werden.

Mindestens folgende Klassen:

* Wissensfrage
* Lernauftrag
* Webauftrag
* Dateioperation
* Statusabfrage
* Governance
* Diagnose
* Memory
* Administration
* Programmierung
* Sonstige

Jede Anfrage erhält genau eine Primärklasse.

---

## 2. KnowledgeAgent

Der KnowledgeAgent beantwortet allgemeine Wissensfragen.

Beispiele:

* Was ist Quantenmechanik?
* Erkläre DNA.
* Was sind binomische Formeln?
* Wie funktioniert Python?

Antwortreihenfolge:

1. vorhandenes Kernwissen
2. gelernte Inhalte
3. Memory
4. lokale Wissensdatenbank
5. Internet (nur falls notwendig)

Nicht jede Wissensfrage darf automatisch eine Internetsuche auslösen.

---

## 3. LearningAgent

Wird aktiviert bei Befehlen wie:

* lerne ...
* studiere ...
* analysiere ...
* merke dir ...
* erweitere dein Wissen ...

Er übergibt Web- oder Dateiquellen an die Lernpipeline.

---

## 4. WebAgent

Nur zuständig für:

* URLs
* Webseiten
* Internetrecherche
* Crawling
* Online-Dokumentationen
* aktuelle Informationen

---

## 5. FileAgent

Nur zuständig für:

* Dateien
* PDFs
* DOCX
* TXT
* Quellcode
* Projektordner
* lokale Dokumente

---

## 6. StatusAgent

Verarbeitet ausschließlich Statusbefehle wie:

* lernstatus
* internetstatus
* webagentstatus
* fileagentstatus
* canonicalenginestatus
* fundamentzyklenstatus

Keine Wissensantworten.

---

## 7. Router-Protokoll

Jede Anfrage wird protokolliert.

Mindestens:

* Zeit
* erkannte Intention
* ausgewählter Agent
* alternative Agenten
* Antwortdauer
* Ergebnis

---

## 8. Explain-Modus

Neuer Befehl:

```
routingstatus
```

Ausgabe:

* erkannte Anfrageklasse
* ausgewählter Agent
* Begründung
* verwendete Quellen
* Fallback-Agenten

---

## 9. Prioritätsregeln

Grundsätzlich gilt:

StatusAgent
→ höchste Priorität

Dateibefehle
→ FileAgent

URLs
→ WebAgent

Lernbefehle
→ LearningAgent

Wissensfragen
→ KnowledgeAgent

Governance
→ CanonicalEngine

Internet nur verwenden,

* wenn aktuelle Informationen erforderlich sind,
* der Benutzer ausdrücklich danach fragt,
* oder das lokale Wissen nicht ausreicht.

---

## 10. Fallback

Kann ein Agent die Anfrage nicht beantworten,

übergibt er kontrolliert an den nächsten geeigneten Agenten.

Keine Endlosschleifen.

Maximal drei Übergaben.

## 13. Governance

Der Router verändert keine Daten.

Er trifft ausschließlich Routing-Entscheidungen.

Alle Entscheidungen werden protokolliert.

Governance-, Foundation- und Canonical-Regeln bleiben unverändert.
