# Codex-Auftrag Teil 15b

## Canonical Self-Presentation API Contracts & SDK (CSPACS) 1.0

### Abschnitt B – SDK, Referenzimplementierungen und Developer Ecosystem

### Ziel

Entwickle für das **Canonical Self-Presentation Framework (CSPF) 1.0** ein vollständig kanonisches SDK- und Integrationssystem, das jede CSPF-konforme Anwendung bei der Erstellung, Validierung und Nutzung von Self-Presentation-Profilen unterstützt.

Das SDK darf keinerlei fachliche Logik verändern.

Es dient ausschließlich als standardisierte Implementierung der zuvor definierten API-Verträge.

---

# Architekturprinzipien

Das SDK muss vollständig deterministisch arbeiten.

Es darf ausschließlich auf den offiziellen API Contracts aufbauen.

Alle SDK-Komponenten müssen automatisch mit zukünftigen API-Versionen kompatibel bleiben.

Alle SDKs besitzen identisches Verhalten.

Unterschiede dürfen ausschließlich durch die jeweilige Programmiersprache entstehen.

# Akzeptanzkriterien

Der Auftrag gilt erst als abgeschlossen, wenn:

* alle offiziellen SDKs dieselben API-Verträge implementieren,
* Builder ausschließlich gültige Self-Presentation-Profile erzeugen,
* sämtliche Validatoren deterministische Ergebnisse liefern,
* alle Serialisierungsformate verlustfrei konvertiert werden,
* Migrationen reproduzierbar und reversibel sind,
* Referenzimplementierungen erfolgreich getestet wurden,
* automatische Dokumentation erzeugt wird,
* alle Komponenten vollständig versioniert, validiert und in das Canonical Self-Presentation Framework (CSPF) 1.0 integriert sind.
