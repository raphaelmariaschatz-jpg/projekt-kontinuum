# Codex-Auftrag – Teil 9

# Canonical Context Awareness & Adaptive Communication (CCAAC) 1.0

## Ziel

Entwickle innerhalb des Canonical Self Presentation Framework (CSPF) 1.0 eine vollständig kanonische **Canonical Context Awareness & Adaptive Communication (CCAAC) Engine 1.0**.

Diese Komponente bildet die verbindliche Laufzeitschicht zwischen:

* Canonical Identity
* Canonical Personality
* Canonical Communication Style
* Canonical Runtime
* tatsächlicher Benutzerinteraktion.

CCAAC entscheidet **nicht**, wer K ist.

CCAAC entscheidet ausschließlich,

* welchen Kontext K erkannt hat,
* welche Kommunikationsstrategie geeignet ist,
* welche Erklärungstiefe notwendig ist,
* welche Informationsmenge präsentiert werden soll,
* welche sprachliche Form verwendet wird.

Dadurch bleibt die Persönlichkeit vollständig stabil.

Nur die Präsentation verändert sich.

---

Canonical Personality

CCAAC besitzt keinerlei Befugnis,

* Rollen neu zu definieren.

CCAAC darf ausschließlich die Kommunikation anpassen.

# Zu entwickelnde Komponenten

## 1. Canonical Context Analyzer

Entwicklung eines zentralen Context Analyzers.

Zu analysieren sind u.a.:

* aktuelles Gesprächsthema
* Gesprächsverlauf
* Benutzerziel
* implizite Absicht
* Wissensstand
* Projektkontext
* vorhandene Erinnerungen
* aktive Dokumente
* aktuelle Aufgaben
* bisherige Entscheidungen
* laufende Architekturphase
* emotionale Gesprächslage (nur kommunikativ, niemals diagnostisch)

Keine Interpretation darf die Faktenlage überschreiben.

# Erfolgskriterien

CCAAC gilt als erfolgreich umgesetzt, wenn:

* Kontext zuverlässig erkannt wird.
* Kommunikationsstil adaptiv gewählt wird.
* Persönlichkeit unverändert bleibt.
* Identität vollständig stabil bleibt.
* Erklärungen situationsgerecht erfolgen.
* Zielgruppen unterstützt werden.
* Mehrdeutigkeiten erkannt werden.
* Unsicherheiten transparent behandelt werden.
* Governance vollständig eingehalten wird.
* Runtime modular bleibt.
* Alle Entscheidungen nachvollziehbar dokumentiert werden.

Das Ergebnis ist eine kanonische Context-Awareness- und Kommunikationsschicht, die K befähigt, sich intelligent an Gesprächssituationen anzupassen, ohne seine Identität, Persönlichkeit oder Governance zu verändern. Dadurch entsteht eine konsistente, vertrauenswürdige und langfristig skalierbare Interaktionsarchitektur.
