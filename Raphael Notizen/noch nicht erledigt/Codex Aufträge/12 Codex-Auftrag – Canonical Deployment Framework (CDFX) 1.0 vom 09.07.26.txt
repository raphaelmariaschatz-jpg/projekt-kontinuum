# Codex-Auftrag – Canonical Deployment Framework (CDFX) 1.0

## Ziel des Auftrags

Bitte prüfe und konzipiere das neue Architekturkonzept:

```text
Canonical Deployment Framework (CDFX) 1.0
```

Projekt Kontinuum soll langfristig in unterschiedlichen Einsatzbereichen verwendet werden können.

Dabei soll **keine getrennte Entwicklung verschiedener Produkte** entstehen.

Stattdessen soll aus einer gemeinsamen Canonical Architecture eine passende Bereitstellungsvariante erzeugt werden.

---

# Grundsatz

Es existiert nur **eine** Canonical Architecture.

Alle späteren Varianten basieren auf demselben Kern.

Unterschiede entstehen ausschließlich durch:

* aktivierte Frameworks
* aktivierte Capabilities
* Governance-Regeln
* Konfigurationen
* Benutzerrollen
* Ressourcen
* Lizenz- bzw. Deployment-Profile

Nicht durch unterschiedliche Quellcodes.

---

# Zielsetzung

Bitte prüfen, ob Projekt Kontinuum künftig kanonische Deployment-Profile unterstützen sollte.

Dabei sollen alle Varianten dieselbe Foundation verwenden.

# Arbeitsauftrag

In dieser Phase erfolgt **keine Implementierung**.

Bitte:

1. analysiere das Konzept
2. prüfe die Einordnung innerhalb der Canonical Architecture
3. definiere den unveränderlichen Canonical Core
4. definiere mögliche Deployment-Profile
5. identifiziere optionale Frameworks
6. prüfe Lizenz- und Rollenmodelle
7. prüfe Erweiterbarkeit für zukünftige Profile
8. erstelle einen kanonischen Implementierungsplan
9. dokumentiere Risiken und offene Fragen

# Erwartetes JSON-Grundschema

Das Framework sollte mindestens enthalten:

* Framework-Informationen
* Version
* Status
* Canonical Core
* Deployment-Profile
* aktivierbare Frameworks
* Rollenmodelle
* Erweiterungspunkte

---

# Leitplanken

Erlaubt:

* Architekturkonzept
* Deployment-Konzept
* Profildefinitionen
* JSON-Schema
* Statusbericht
* Glossar-Ergänzung
* Implementierungsplan

Nicht erlaubt:

* Trennung der Codebasis
* Duplizierung von Frameworks
* produktspezifische Forks
* Runtime-Migration
* Änderungen produktiver Komponenten

---

# Entscheidungskriterium

Das Canonical Deployment Framework soll nur dann als GO bewertet werden, wenn es:

* eine gemeinsame Codebasis erhält,
* den Canonical Core unverändert lässt,
* Deployment ausschließlich über Profile und Konfigurationen steuert,
* zukünftige Varianten ohne Architekturbruch ermöglicht,
* langfristig wartbar bleibt.

---

# Erwartetes Ergebnis

Bitte liefere:

1. Bestandsanalyse
2. Definition des Canonical Core
3. Vorschlag für Deployment-Profile
4. Aktivierbare Frameworks und Capabilities
5. Schnittstellenübersicht
6. Vorschlag für Dokumente und JSON-Strukturen
7. Risiken und offene Fragen
8. Empfehlung: GO / NO-GO / SPÄTER
9. Kanonischen Implementierungsplan
10. Statusbericht unter `31_reports`, falls umgesetzt.

Ziel ist ein universeller Bereitstellungsrahmen, mit dem Projekt Kontinuum aus einer einzigen Canonical Architecture verschiedene Einsatzprofile für Privatanwender, Unternehmen und Forschung bereitstellen kann, ohne den gemeinsamen Kern oder die Wartbarkeit zu gefährden.
