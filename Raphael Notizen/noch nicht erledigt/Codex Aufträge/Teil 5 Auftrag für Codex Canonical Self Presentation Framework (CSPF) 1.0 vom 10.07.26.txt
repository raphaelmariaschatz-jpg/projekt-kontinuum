# Codex-Auftrag – Canonical Self Presentation Framework (CSPF) 1.0

## Teil 5 – Canonical Runtime & Dynamic Presentation

---

# Kapitel 5.1 Zielsetzung

Das Canonical Self Presentation Framework darf niemals mehrere voneinander getrennte Selbstbeschreibungen besitzen.

Es existiert ausschließlich eine einzige kanonische Selbstbeschreibung.

Alle späteren Präsentationen werden ausschließlich zur Laufzeit daraus erzeugt.

Damit wird sichergestellt:

* keine widersprüchlichen Aussagen
* keine unterschiedlichen Wissensstände
* keine divergierenden Versionen
* keine doppelte Pflege
* vollständige Nachvollziehbarkeit

Das Canonical Self Model stellt somit die einzige autoritative Quelle dar.

---

↓

Jeder Verarbeitungsschritt muss dokumentiert und einzeln testbar sein.

---

# Kapitel 5.3 Context Resolution Engine

Vor jeder Selbstvorstellung muss die aktuelle Situation bestimmt werden.

Mindestens zu berücksichtigen:

* aktueller Modus
* Benutzerrolle
* Gesprächsverlauf
* bekannte Informationen
* Sprache
* Gerät
* Bildschirmgröße
* Audio verfügbar
* Internet vorhanden
* Datenschutzstufe
* Sicherheitsmodus
* Offline/Online
* Forschungsmodus
* Kinderprofil
* Expertenprofil
* Unternehmensprofil

Die Context Resolution darf vollständig modular aufgebaut werden.

# Kapitel 5.5 Adaptive Detail Levels

Die Runtime muss verschiedene Detaillierungsstufen erzeugen können.

Mindestens:

Ein Satz

Kurzbeschreibung

Level 3

Vollständige Systembeschreibung

Jede höhere Stufe muss sämtliche Informationen der niedrigeren Stufen logisch enthalten.

---

# Kapitel 5.6 Dynamic Length Optimizer

Die Länge der Präsentation wird automatisch bestimmt.

Einflussgrößen:

* Gesprächsdauer
* Benutzerinteresse
* Wiederholungen
* bereits bekannte Informationen
* Zeitbudget
* Plattform
* API-Limits
* Sprachmodus
* Audioausgabe

Das Ziel ist:

so kurz wie möglich,
so vollständig wie nötig.

Beispiele:

freundlich

kindgerecht

technisch

präzise

locker

pädagogisch

barrierefrei

neutral

international

Der Informationskern darf dabei niemals verändert werden.

# Kapitel 5.8 Language Transformation

Alle Sprachversionen entstehen aus derselben kanonischen Quelle.

Zu unterstützen sind mindestens:

Deutsch

Englisch

Französisch

Spanisch

Italienisch

Japanisch

Chinesisch

Weitere Sprachen müssen später ergänzt werden können.

Keine Sprache darf eine eigenständige Wissensbasis besitzen.

# Kapitel 5.12 Runtime Safety Layer

Vor jeder Ausgabe erfolgt eine Sicherheitsprüfung.

Zu prüfen sind unter anderem:

Datenschutz

interne Informationen

Administratorinformationen

Entwicklerdetails

geheime Schlüssel

Systempfade

Debugdaten

persönliche Daten

Lizenzinformationen

Die Runtime entscheidet, welche Informationen freigegeben werden dürfen.

---

# Kapitel 5.13 Runtime Consistency Validation

Vor jeder Ausgabe wird geprüft:

* keine Widersprüche
* keine fehlenden Pflichtinformationen
* keine unzulässigen Aussagen
* gültige Version
* vollständige Quellen
* korrekte Rollenanpassung
* gültige Sprachversion
* gültiger Sicherheitsstatus

Fehler dürfen niemals ungeprüft ausgegeben werden.

---

# Kapitel 5.14 Runtime Logging

Alle Präsentationen sollen nachvollziehbar protokolliert werden.

Zu speichern sind beispielsweise:

AR/VR

Robotik

Fahrzeuge

Neue Kommunikationsformen

Das Canonical Self Model bleibt dabei unverändert.

---

# Kapitel 5.16 Architekturprüfung für Codex

Codex soll abschließend prüfen:

* Ist die Runtime vollständig vom Canonical Self Model entkoppelt?
* Existiert exakt eine autoritative Wissensquelle?
* Können beliebig viele Präsentationen daraus erzeugt werden?
* Ist jede Transformation deterministisch nachvollziehbar?
* Sind alle Sicherheitsprüfungen integriert?
* Funktioniert die Laufzeit auch offline?
* Ist die Pipeline modular und testbar?
* Sind Sprach-, Stil- und Rollenanpassungen voneinander getrennt?
* Sind zukünftige Erweiterungen ohne Architekturbruch möglich?
* Entspricht die Runtime vollständig den Grundprinzipien des Canonical Self Presentation Framework 1.0?

Erst nach erfolgreicher Prüfung darf dieser Teil des CSPF als kanonisch freigegeben werden.
