# Codex-Auftrag – Canonical Self Presentation Framework (CSPF) 1.0

## Teil 4 von 20

# Phase 4 – Canonical Validation, Governance & Trust Framework

### Ziel

Entwickle den vierten Bestandteil des Canonical Self Presentation Framework (CSPF) 1.0.

Dieser Abschnitt definiert sämtliche Mechanismen zur Sicherstellung der Vertrauenswürdigkeit einer Selbstpräsentation.

Eine Selbstvorstellung darf niemals lediglich erzeugt werden.

Sie muss jederzeit nachvollziehbar, überprüfbar, konsistent und auditierbar sein.

---

# 4.1 Canonical Validation Layer (CVL)

Entwickle einen eigenständigen Validation Layer.

Dieser prüft vor jeder Ausgabe:

* Vollständigkeit
* Konsistenz
* Berechtigungen
* Freigaben
* Gültigkeit
* Version
* Integrität
* Signatur
* Richtlinien

Nur vollständig validierte Präsentationen dürfen ausgegeben werden.

# 4.6 Canonical Truth Check

Implementiere eine Truth Engine.

Regeln:

Keine Fantasie

Keine Spekulation

Keine erfundenen Fähigkeiten

Keine erfundenen Versionen

Keine erfundenen Module

Keine Behauptungen ohne Nachweis

Lieber weniger Informationen
als falsche Informationen.

# 4.8 Contradiction Detection

Das CSPF muss Widersprüche erkennen.

Beispiele:

Version passt nicht.

Capability existiert nicht.

Funktion deaktiviert.

Lizenz fehlt.

Authentifizierung fehlt.

Persona widerspricht Rolle.

Kontext widerspricht Richtlinie.

Mehrsprachige Inhalte widersprechen sich.

Bei Widersprüchen:

Ausgabe stoppen.

Fehler protokollieren.

---

## Ziel dieses Abschnitts

Nach Abschluss von Teil 4 besitzt das Canonical Self Presentation Framework eine vollständige Vertrauens- und Governance-Schicht. Jede Selbstpräsentation wird nicht nur erzeugt, sondern vor ihrer Ausgabe anhand nachvollziehbarer Regeln validiert, sicherheitsgeprüft, dokumentiert und revisionssicher protokolliert. Das CSPF wird damit zu einer überprüfbaren und langfristig belastbaren Grundlage für die Selbstdarstellung von K in unterschiedlichen Rollen, Kontexten und Entwicklungsständen.
