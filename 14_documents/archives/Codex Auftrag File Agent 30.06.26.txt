Codex Auftrag File Agent 30.06.26

Codex-Auftrag: File Agent 1.0 – Dateien hochladen, öffnen, lesen und lernen, falls gewünscht auch abspeichern

Ziel:
Bitte implementiere einen eigenständigen FileAgent für Kontinuum 34.1+, der lokale und hochgeladene Dateien kontrolliert öffnen, lesen, analysieren und als Lernquelle an das bestehende Lern-/Memory-/Knowledge-System übergeben kann.

Ausgangsproblem:
Kontinuum braucht neben dem WebAgent einen FileAgent, damit Raphael Dateien gezielt bereitstellen kann und K daraus wirklich lernt, statt Dateinamen oder Befehle nur lokal zu speichern.

Pflichtfunktionen:

1. Datei-Erkennung
- Erkenne Nutzerbefehle wie:
  - lies diese Datei
  - lerne aus dieser Datei
  - öffne diese Datei
  - analysiere diese Datei
  - importiere diese Datei ins Wissen
  - lerne aus allen Dateien in diesem Ordner
- Unterscheide Datei-/Ordnerpfade klar von normalen Suchanfragen.

2. Unterstützte Dateitypen
Mindestens:
- .txt
- .md
- .json
- .csv
- .html
- .pdf
- .docx
- .xlsx
- .py
- .js
- .css
- .html
- .log
- .epub
- .azw
- .azw3
- .kfx

3. Sichere Dateiöffnung
- Nur erlaubte Projekt- oder Upload-Verzeichnisse lesen.
- Keine Systemordner ohne Freigabe.
- Keine ausführbaren Dateien starten.
- Große Dateien begrenzen und protokollieren.
- Dateipfad, Dateigröße, Hash und Zeitstempel erfassen.

4. Inhalts-Extraktion
- Text aus unterstützten Formaten extrahieren.
- Bei Code-Dateien zusätzlich Sprache, Funktionen, Klassen und Kommentare erkennen.
- Bei Tabellen erste Zeilen, Spaltennamen und Struktur erkennen.
- Bei PDF/DOCX Überschriften, Absätze und Tabellen extrahieren, soweit möglich.

5. Lernübergabe
- Extrahierte Inhalte an das bestehende Lernsystem übergeben.
- Speichern:
  - Dateiname
  - Pfad
  - Dateityp
  - Thema
  - Kurzinhalt
  - Volltext-Auszug
  - Hash
  - Importdatum
  - Lernstatus
- Doppelimporte durch Datei-Hash verhindern.

6. Ordner-Import
- Neuer Befehl:
  lerne aus Ordner <Pfad>
- Rekursiv optional, Standard nicht rekursiv.
- Begrenzung:
  - max_files Standard: 50
  - max_file_size Standard: 400 MB
  - nur erlaubte Endungen
- Ergebnisbericht mit gelesenen, übersprungenen und fehlerhaften Dateien.

7. FileAgent-Status
Neuer Befehl:
  fileagentstatus

Ausgabe:
- FileAgent aktiv: ja/nein
- erlaubte Verzeichnisse
- unterstützte Dateitypen
- zuletzt gelesene Datei
- Anzahl importierter Dateien
- letzte Fehler
- Duplikate erkannt
- Lernübergabe erfolgreich: ja/nein

8. GUI-Integration
Bitte in der GUI ergänzen:
- Button „Datei öffnen“
- Button „Datei lernen“
- Button „Ordner lernen“
- Statusanzeige im Aktivitätsfenster
- klare Ausgabe, welche Datei gelesen und was gelernt wurde.

9. Abnahmetests
Bitte Tests ergänzen für:
- lies Datei test.txt
- lerne aus Datei beispiel.md
- analysiere Datei beispiel.py
- importiere PDF als Lernquelle
- lerne aus Ordner 14_documents
- fileagentstatus
- Duplikatimport derselben Datei
- nicht erlaubter Pfad
- unbekannter Dateityp

10. Governance
- Keine Dateien löschen oder verändern.
- FileAgent arbeitet zunächst read-only.
- Jeder Import wird protokolliert.
- Jede gelernte Datei erhält Quelle, Hash und Zeitstempel.
- Bestehende Canonical-/Foundation-/Governance-Regeln nicht umgehen.

Ergebnis:
Bitte FileAgent 1.0 zunächst diagnostisch/read-only integrieren und danach aktivieren.

Bitte Abschlussbericht erstellen:
14_documents/file_agent_1_0_umsetzung_<datum>.md
