Codex-Auftrag: VisionAgent für Projekt Kontinuum integrieren

Ziel:
Bitte einen neuen VisionAgenten in Projekt Kontinuum integrieren, der Bilder analysieren, Bildinhalte beschreiben und erkannte Informationen kanonisch an das bestehende System anbinden kann.

Grundsatz:
Der VisionAgent darf zunächst read-only arbeiten. Er soll Bilder lesen, analysieren, beschreiben und Ergebnisse strukturiert zurückgeben, aber keine Originaldateien verändern.

Aufgaben:

1. Bestehende Architektur prüfen

* vorhandene Agentenstruktur analysieren
* bestehende Registry-/Router-/Agenten-Patterns verwenden
* keine Parallelarchitektur erzeugen
* VisionAgent sauber in die vorhandene Agent Registry eintragen

2. VisionAgent als eigenes Modul erstellen
   Vorgeschlagener Name:

* VisionAgent
* Datei z. B. in passendem Agenten-/Core-Verzeichnis gemäß aktueller Projektstruktur

Funktionen:

* Bilddateien laden
* unterstützte Formate erkennen: png, jpg, jpeg, webp, bmp, gif, tiff
* einfache Bildmetadaten auslesen
* Bildgröße, Dateityp, Farbraum, Dateipfad erfassen
* sichere Fehlerbehandlung bei beschädigten oder nicht unterstützten Dateien

3. Analysefunktion vorbereiten
   Der VisionAgent soll eine zentrale Methode erhalten, z. B.:

analyze_image(path: str) -> dict

Rückgabe als strukturierter Dict/JSON-Block mit:

* status
* source_path
* file_exists
* format
* width
* height
* mode/color_space
* summary
* warnings
* errors
* timestamp

Falls noch kein echtes KI-Vision-Modell angebunden ist:

* bitte Platzhalter-/Fallback-Analyse implementieren
* klar kennzeichnen: "vision_model_available": false
* keine erfundenen Bildinhalte ausgeben
* nur technisch belegbare Informationen zurückgeben

4. Router-Anbindung
   Neue Befehle ermöglichen, z. B.:

* "analysiere bild <pfad>"
* "beschreibe bild <pfad>"
* "vision <pfad>"
* "lies bild <pfad>"

Der Request Router soll solche Anfragen an den VisionAgent weiterleiten.

5. Quellen- und Provenienzlogik
   Vision-Ergebnisse müssen die Quelle eindeutig ausweisen:

* lokale Datei: ja
* Dateipfad
* Analysezeitpunkt
* verwendeter Agent
* verwendetes Verfahren
* ob echte Bildinhaltserkennung oder nur Metadatenanalyse durchgeführt wurde

6. Knowledge-/Memory-Anbindung vorbereiten
   Vision-Ergebnisse sollen optional später in Memory/Knowledge Graph übernommen werden können, aber zunächst nicht automatisch dauerhaft speichern.

Bitte vorbereiten:

* exportierbares Analyseergebnis als JSON
* klare Trennung zwischen Analyse und Speicherung
* keine automatische Memory-Veränderung ohne ausdrücklichen Befehl

7. Sicherheit und Pfadfreigabe
   Der VisionAgent muss bestehende Pfadfreigabe- und Sicherheitsregeln respektieren.
   Keine Analyse außerhalb freigegebener Projekt-/Datenpfade.
   Bei nicht freigegebenem Pfad:

* sauber ablehnen
* verständliche Fehlermeldung ausgeben
* keine Datei öffnen

8. GUI-Vorbereitung
   Falls die GUI bereits Agentenstatus oder Quellenausgabe unterstützt:

* VisionAgent dort sichtbar machen
* Quelle "lokale Bilddatei" anzeigen
* Analysemodus anzeigen
* Fehler verständlich ausgeben

9. Tests erstellen
   Bitte Tests für folgende Fälle ergänzen:

* gültiges PNG
* gültiges JPG
* nicht vorhandene Datei
* nicht unterstütztes Format
* beschädigte Datei
* nicht freigegebener Pfad
* Router-Befehl "analysiere bild <pfad>"

10. Abschlussbericht erstellen
    Bitte nach Umsetzung einen kurzen Bericht erzeugen mit:

* geänderten Dateien
* neu angelegten Dateien
* Registry-/Router-Anbindung
* unterstützten Bildformaten
* Testergebnissen
* bekannten Einschränkungen
* nächstem sinnvollen Ausbauschritt

Wichtig:
Keine bestehenden Agenten beschädigen.
Keine Originalbilder verändern.
Keine Bildinhalte erfinden, wenn kein echtes Vision-Modell angebunden ist.
Read-only zuerst.
Kanonische Integration vor Funktionsfülle.
